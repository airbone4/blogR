R的基本的輸出函數

大概有這幾種：

  • print()
  • cat()
  • message()
  • warning()
  • stop()

前兩個輸出到標準輸出裝置(standard output)，後三個會顯示在標準錯誤裝置上(standard error)。 如果不想要顯示訊息,

  • 是stderr 類的輸出函數,可以利用 suppressWarnings(), suppressMessages(), suppressPackageStartupMessages()。
  • stdin 類則使用 sink() or capture.output()

另外有關Rmarkdown: knitr 提供的選項,像是 error = F, message = F, and warning = F.

函數說明

  • print() - 無法連結字串，會帶有[1]，例如：

    print(paste("Hello","World!"))

[1] "Hello World!"

  • cat() 帶有類似paste()的參數，sep=" "，所以連結字串比較方便，但是必須自行寫換行符號\n。

    cat("Hello!\n")
#> Hello!
#>                      # 🏷👏這是空白行,動應字串中的最後一個\n1  
cat("Hello","World!\n") # 🏷👏意思是說碰到\n就換行
#> Hello World!
#>                      
cat("Hello","World!\n", sep = "")
#> HelloWorld!  # # 🏷👏全部擠在一起,而且\n無用

  • message() 帶紅字的輸出，基本上是到標準錯誤裝置。

    message("Hello ","World!") # Note the space after Hello
#> "Hello World!"
    • 更方便的是能用來配合tryCatch()e.g. 
      tryCatch(message("hello\n"), message=function(e){cat("goodbye\n")})
#> goodbye

  • warning() : 我們自訂的函數檢查到參數(可能是類別)可能導致語意錯誤的訊息。

  • stop(): 錯誤訊息，再繼續計算也是錯的。

另外,在套件開發中的函數,可以盡量少用print()。如果要輸出的函數屬於S3物件的print.*(),那麼就使用cat(),其他狀況則使用message()。warning(),stop()

基本輸入和輸出

  • print()
  • cat()
  • sprintf()
a<-"hello\nR" # 🏷👏也可以是單引號'hello\n'
print(a) # 也可以不用先指派變數,而直接 print("hello")
#> [1] "hello\nR"
cat(a)
#> hello
#> R

print(),cat()雖然在上面的例子看起來一樣。但是初學者推薦使用print()。

  1. cat() 認識字串中的換行符號\n, print() 不認識
  2. 如果有多個字串,使用cat()的時候,可以設定cat(..., sep="\n")。
補充

cat()通常是在檔案處理的時候。在C類的語言中,鍵盤和螢幕都屬於檔案-特殊檔案,一種只能讀出,一種只能寫入,抽象觀念的描述是串流。 使用範例如下:

sink("foobar.txt") # <!--html_preserve--><span  class="codenote nopre">🏷<span   class="tooltiptext" >從這個指令開始,所有流向螢幕的輸出都輸出到檔案foobar.txt</span></span><!--/html_preserve-->
cat('"foo"\n')
cat('"bar"')
sink() #<!--html_preserve--><span  class="codenote nopre">🏷<span   class="tooltiptext" >結束</span></span><!--/html_preserve-->

檔案內容是:

"foo"
"bar"
  • 一行指令完成上述程式碼 
      cat('"foo"', '"bar"', file="foobar.txt", sep="\n")
  • 如果檔案已經存在,還要添加,那麼就加上參數append=T 
      cat('"foo"', file="foobar.txt", append=TRUE)
  cat('"bar"', file="foobar.txt", append=TRUE)
