在R中,options()經常用來設定總體變數,例如套件blogdown可以設定options(blogdown.method = 'custom')。

如果想要知道目前的設定情況,可以下指令options()，例如

options() # 🏷註解這是在視窗下執行的結果

如上可以知道有一大串列表,如果我只想知道套件blogdown的設定列表，要怎樣做? 也就是怎樣篩選出我要的設定?

  • 解法1:利用grep(),grepl()
  • 解法2:sapply , 也是用到grep() 不能用 %in%

可以參考這裡

  • 解法1
olist<-options() # list
okey<-names(olist) # vector 
# "blogdown" %in% okey
idx<-grep("blogdown",okey)
olist[idx]

#> $blogdown.method
#> [1] "markdown"
#> 
#> $blogdown.serve_site.startup
#> [1] FALSE
  • 解法2:看一下為甚麼不能用 %in%
olist<-options() # list
okey<-names(olist) # vector 

rst <- sapply(okey,function(item){"blogdown" %in% unlist(strsplit(item,"\\."))  })
olist[rst]

#> $blogdown.method
#> [1] "markdown"
#> 
#> $blogdown.serve_site.startup
#> [1] FALSE

解釋這行: rst <- sapply(okey,function(item){"blogdown" %in% unlist(strsplit(item,"\\.")) })

  • \\.而不是只有.
  • strsplit()執行的結果是一個list而分離的元素在第一個node所以,用unlist
  • %in%的用法
v=c("1.2","1.3")
"1" %in% v

#> [1] FALSE

⛳註解 ❓在主控台上的比較多,但是在rmarkdown chunk只有看到幾個,因為在rmarkdown中沒有載入blogdown套件。

練習:找出目前的語系

利用l10n_info() 找出目前的語系,而不是一連串真假值。 
l10n_info()[2:3][(l10n_info()[2:3])==TRUE]
