簡單的函數範例

  • 求平均,標準差

    mymean<-function(x){
  sum(x)/length(x) #🏷👏最後一個執行的express其結果會被回傳
}
std <-function(x){
  sd(x)
}

data<-runif(10)

mymean(data)
#> [1] 0.3969562
mean(data) # 🏷👏比較原來的mean
#> [1] 0.3969562
std(data)
#> [1] 0.2328856
sd(data) # 🏷👏比較原來的sd
#> [1] 0.2328856

  • 另一範例

    power <- function(exponent) {
  function(x) x^exponent
}

square <- power(2)
square(2) # -> [1] 4
square(4) # -> [1] 16

cube <- power(3)
cube(2) # -> [1] 8
cube(4) # -> [1] 64

    myfun<-  function (x, na.rm = TRUE) #🏷👏第二個參數有預設值
{
    xna <- is.na(x)
    if (any(xna)) {
        if (na.rm) 
            x <- x[!xna]
        else return(rep.int(NA, 5))
    }
    x <- sort(x) #🏷👏排序
    n <- length(x)
    if (n == 0) 
        rep.int(NA, 5)
    else {
        n4 <- floor((n + 3)/2)/2
        d <- c(1, n4, (n + 1)/2, n + 1 - n4, n)
        0.5 * (x[floor(d)] + x[ceiling(d)])
    }
}

全局變數和局部變數

  • 在函數中使用的變數,是局部變數 例如本例中的y。

    x <- 2 
ls() #🏷👏全局變數x，✔可以在工作區間看到x
#> [1] "data"   "mymean" "std"    "x"
g <- function() { #🏷👏全局變數g，✔可以在工作區間看到
    y <- 1  #🏷👏局部變數y，❌在全局區看不到,只有在目前的函數區可以看到
    c(x, y) #🏷👏在函數中使用全局變數x
}
g()
#> [1] 2 1
y # 🏷🐞會出現錯誤
#> Error in eval(expr, envir, enclos): 找不到物件 'y'
ls() # 工作區出現x,g,但是沒有y
#> [1] "data"   "g"      "mymean" "std"    "x"
rm(x, g)
ls()
#> [1] "data"   "mymean" "std"

  • 局部函數工作區中,想要更改全局變數

    x<-3
f1<-function (){
  x<-4+x # 🏷👏不會更改全局變數x,只是局部變數,的確是7
}
f1()
x #🏷👏仍然是3,看起來是R的參數傳遞方法是傳值而不是傳址
#> [1] 3

  • 上述程式碼如果要在函數中更改全局變數,則

    x<-3
f1<-function (){
  x<<-4+x #🏷👏這會更改全局變數x✔️
}
f1()
x # 7
#> [1] 7

  • 串列呢

    x<-list(1,2,3)
f1<-function (){
  x[[4]]<<-"a4" # 🏷👏✔️
}
f1()
x # 7
#> [[1]]
#> [1] 1
#> 
#> [[2]]
#> [1] 2
#> 
#> [[3]]
#> [1] 3
#> 
#> [[4]]
#> [1] "a4"

parent.frame

parentls <- function() {
  ls(envir=parent.frame())
}

a<-function() {
    x <- 5
    parentls()
}

b <- function() {
    z <- 10
    parentls()
 }

a()
#> [1] "x"
b()
#> [1] "z"

3個逗點 (dots)

3個逗點可以利用?dots查詢。

範例

f <- function(x, ...)
{
  dots <- list(...)                   #1
  if(length(dots) == 0) return(NULL) 
  cat("The arguments in ... are\n")
  print(x)
  f(...)                              #2
}

f(1,2,3,"a", list("monkey"))
#> The arguments in ... are
#> [1] 1
#> The arguments in ... are
#> [1] 2
#> The arguments in ... are
#> [1] 3
#> The arguments in ... are
#> [1] "a"
#> NULL

operator %>%

%>% 不是R基礎套件 而是定義在套件 magrittr (CRAN) 之中 且常跟 dplyr (CRAN)搭配。

意思是把左邊(LHS)當成右邊(RHS)函數中的第一個參數。

例如下面的例子:

library(magrittr)
iris %>% head() #🏷👏 iris %>% head() 相當於 head(iris)
Sepal.Length Sepal.Width Petal.Length Petal.Width Species
1          5.1         3.5          1.4         0.2  setosa
2          4.9         3.0          1.4         0.2  setosa
3          4.7         3.2          1.3         0.2  setosa
4          4.6         3.1          1.5         0.2  setosa
5          5.0         3.6          1.4         0.2  setosa
6          5.4         3.9          1.7         0.4  setosa

為甚麼需要這樣用,下面是一個例子

iris %>% head() %>% summary()

類似的觀念,

iris %>% head() %>% summary() # 等同於summary(head(iris)).

也就是說,避免了使用巢狀呼叫。

closure

ox<-function (arg=3)  {
  x<-0
  o1<-function(){
    x<<-x+arg;
  } 
  list(go=o1)
}

obj<-ox(4)
obj$go()

example of closure


make.accumulator<-function(){
             a<-0
             function(x) {
                  a<<-a+x
                   a
            }
}

f<-make.accumulator()
f(1)
#> [1] 1
f(1)
#> [1] 2
f(1)
#> [1] 3

❓ 利用上述觀念建立一個計數器。

💡 例如產生的檔案個數不定,但是想要每次開始執行的時候都有一個固定編號。

new_counter <- function() {
  i <- 0
  function() {
    # do something useful, then ...
    i <<- i + 1
    i
  }
}

source

自己的函數存成檔案(例如名稱為myfun.R)以後,如何呼叫? 利用source("myfun.R") 可以載入記憶體中,此時在整體工作環境中可以看到函數定義。

問題是,下次我又繼續同一個專案的時候呢,這些我已經定義過的函數能不能續續使用?有下列幾種方法

  1. 離開專案的時候,如果儲存工作環境(系統問你是不是save workspace to .Rdata?),如果回答是,則下次可以不用source(),直接可以使用。
  2. 在.Rprofile 加入一行source("myfun.R")
  3. 但是在RMD檔案中,因為每次都是新session因此,必須要先找地方source(例如在第一個chunk)。

Base objects vs OO objects

辨別變數是基礎型別或是物件型別，可以簡單利用函數： is.object():

基礎物件base object:

is.object(1:10)
#> [1] FALSE

物件導向物件 OO object

is.object(mtcars)
#> [1] TRUE

主要的區別在於基本物件沒有class 這個attribute

attr(1:10, "class") #  NULL

attr(mtcars, "class") # [1] "data.frame"
