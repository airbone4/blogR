分解、合成檔案路徑

合成路徑

workingdir <- "work"
projectdir <- "demo"
datadir    <- "data"
y<-file.path(workingdir, projectdir, datadir)
y
#> [1] "work/demo/data"
typeof(y)
#> [1] "character"

上面是windows 10系統上的結果,在 Linux-based 的作業系統執行結果一樣,注意斜線都一樣。

利用函數 file.path()合成檔案路徑，其分隔是斜線,而不是反斜線 (例如for example C:/Program Files/R/R-3.5.1)?

路徑分解

  • dirname() :分解路徑用，例如＂c:/temp/text.txt" 得到"c:/temp"，但是"c:/temp/","c:/temp", "c:/temp.txt"都得到"c:/"。😒🏷
  • basename()
dirname("c:\\temp\\demo.txt")
#> [1] "c:/temp"
basename("c:\\temp\\demo.txt")
#> [1] "demo.txt"

dirname("c:/temp/") #🏷
#> [1] "c:/"
dirname("c:/temp") #🏷
#> [1] "c:/"

檔案系統相關操作

  • dir():包括資料夾、檔案名稱
  • list.files() : 僅檔案
  • list.dirs():僅資料夾

判別是否目標檔案或資料夾已經存在(我們不希望刪除或更動已經存在的檔案):

  • file.exists("xx.txt"): 是否存在xx.txt

  • dir.exists("xx"): 是否存在子目錄xx

  • 工作區相關

    • getwd()
    • setwd()
    • .libPaths() : 目前套件安裝的地方。
    • R.Home() : 目前R安裝的地方,例如"C:/PROGRA~1/R/R-41~1.1"

範例

currentwd<-getwd()
setwd("~") # 🏷
getwd()
#> [1] "C:/Users/you/Documents"
setwd(currentwd)

path.expand("~") # 🏷

R.home()
libPaths()
list.files(R.home()) #:可以列出R的安裝的位置,然後list.files()列出檔案

list.dirs(R.home("doc"))
list.dirs(R.home("doc"), full.names = FALSE)

## 在使用者指定的目錄,列出 檔案的開頭是 a到l 或 r。
dir("../..", pattern = "^[a-lr]", full.names = TRUE, ignore.case = TRUE)

練習

按照上面的結果,如果我要判斷我拿到是檔案而不是資料夾,怎麼做? 
file.exists(f) && !dir.exists(f)

問題: 利用.libPaths() 、 list.files()問題:列出所有套件 
list.files(.libPaths())

操作檔案系統

  • 暫時子目錄的路徑tempdir()，暫時檔案tempfile()

    • 函數 tempfile() 不是真正的建立新檔案,而是在目前的r session中隨機產生唯一檔案名稱，檔案位置預設是在暫時子目錄中。
    • mydirname <- tempfile(pattern = "mydir")
mydirname

#> C:\\Users\\使用者名稱\\AppData\\Local\\Temp\\RtmpSg2qRA\\mydir326845a02140

  • 函數 file.exists() 可以用來知道檔案是否存在,

  • 函數 dir() 用來列出子目錄中的檔案,例如dir(tempdir())

  • 範例
    s<-"C:\\Users\\使用者名稱\\AppData\\Local\\Temp"
dir(s)

R系統檔案列表

如果要觀察安裝套件的檔案位置,可以使用指令system.file(),這個指令可以列出內建某個套建的全路徑。例如,

system.file(package = "stats")
# > [1] "C:/PROGRA~1/R/R-3.5.1/library/stats"
system.file() 🏷
# > [1] "C:/PROGRA~1/R/R-41~1.1/library/base" 
.libPaths() 🏷
# > [1] "C:/Users/you/Documents/R/win-library/4.1"
# > [2] "C:/Program Files/R/R-4.1.1/library"

system.file()的結果是一個目錄,因此可以利用dir()列出套件stats中,所有的檔案

dir(system.file(package = "stats"))

#>  [1] "COPYRIGHTS.modreg" "demo"              "DESCRIPTION"    
#>  [4] "help"              "html"              "INDEX"          
#>  [7] "libs"              "Meta"              "NAMESPACE"      
#>  [10] "R"                 "SOURCES.ts"

上面可以看到套件中包含子目錄demo，如果還要查看demo中的檔案內容:

dir(system.file("demo", package = "stats"))

#> [1] "glm.vr.R" "lm.glm.R" "nlm.R"    "smooth.R"

上面只有檔案名稱,如果要看全路徑，可以利用參數full.names:

dir(system.file("demo", package = "stats"), full.names = TRUE)

#> [1] "/usr/lib64/R/library/stats/demo/glm.vr.R"
#> [2] "/usr/lib64/R/library/stats/demo/lm.glm.R"
#> [3] "/usr/lib64/R/library/stats/demo/nlm.R"   
#> [4] "/usr/lib64/R/library/stats/demo/smooth.R"

# only copy files/dirs if they exist
file.copy2 = function(from, to, ...) {
  i = file.exists(from); from = from[i]
  if (length(from) == 0) return()
  if (length(to) > 1) {
    to = to[i]
    if (length(unique(to)) == 1) to = unique(to)
  }
  if (length(to) == 1) {
    file.copy(from, to, ...)
  } else mapply(file.copy, from, to, ...)
}

dir_rename = function(from, to, clean = FALSE) {
  if (!dir_exists(from)) return()
  if (clean) unlink(to, recursive = TRUE)
  dir_create(dirname(to))
  # I don't know why file.rename() might fail, but if it fails, fall back to
  # file.copy(): https://github.com/rstudio/blogdown/issues/232
  suppressWarnings(file.rename(from, to)) || {
    file.copy(from, dirname(to), recursive = TRUE) && unlink(from, recursive = TRUE)
  }
}

dirs_rename = function(from, to, ...) {
  n = length(from); if (n == 0) return()
  if (length(to) != n) stop(
    'The number of source dirs must be equal to the number of target dirs'
  )
  for (i in seq_len(n)) dir_rename(from[i], to[i], ...)
}

# relative path with '/' as the path separator
rel_path = function(x) {
  xfun::relative_path(xfun::normalize_path(x))
}

# change the default of full.names and recursive in list.files() because these
# values are used much more frequently than the original defaults
list_files = function(..., full.names = TRUE, recursive = TRUE) {
  list.files(..., full.names = full.names, recursive = recursive, no.. = TRUE)
}
