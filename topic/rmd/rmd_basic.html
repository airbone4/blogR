我比較常用的格式

這裡稍微解釋一下RMD文件表頭中的yaml:

---
title: "處理2003(92R)變數AB v1"
author: "lin"
date: "29 十月, 2020, 12:25"
header-includes:
- \usepackage{xeCJK}
- \setCJKmainfont{標楷體}
output:
  html_document:  
    toc: yes
    toc_float:
      collapsed: false
#     smooth_scroll: true        
#    df_print: paged
    highlight: tango  
    css: ["adv-r.css","mystyle.css"]  
  word_document:
    toc: yes
  pdf_document: 
    latex_engine: xelatex 
    toc: yes
knit:   myknit 
---
  • 上面的word_document,其實對應 rmarkdown::word_document()
  • 而下面的 toc: yes 是要傳入上面函數的參數.可以對照?rmarkdown::word_document()

例如,如果我想要把我比較常用的word 模板,當成RMD 產生WORD文件的模板,可以 步驟1:先利用RMarkdown產生word 文件,然後在yml的參數中指定:

---
...
word_document:
  reference_docx: myword.docx
---

指令 rmarkdown::render_site() 用來建立網站。這個指令需要,一個檔案 _site.yml,內容如下:

name: "my-website"
navbar:
  title: "My Website"
  left:
    - text: "Home"
      href: index.html
    - text: "About"
      href: about.html

同時檔案index.html 是第一個網站的入門網頁。

book

指令 bookdown::render_book() 需要的檔案有_bookdown.yml'_output.yml’。

_bookdown.yml

The _bookdown.yml 那些檔案要被包含進來。

rmd_files: ["index.Rmd", "02-literature.Rmd", "01-intro.Rmd"]

_output.yml

The _output.yml 用來指定如何輸出,例如HTML和PDF的格式。範例:

bookdown::gitbook:
  lib_dir: assets
  split_by: section
  config:
    toolbar:
      position: static
bookdown::pdf_book:
  keep_tex: yes
bookdown::html_book:
  css: toc.css

splitby: 後面可以是RMD,每個RMD一個HTML檔案。

index.Rmd

通常是一系列文章裡面的第一個文件。其他rmd文件的yml 頭部都可以統一放在這裡。

下面是一些選項,可以自行添加到上面:

bibliography: [book.bib, packages.bib]
biblio-style: apalike
link-citations: yes
github-repo: rstudio/bookdown-demo

bibliography 暫時沒有參考文件檔,所以我沒有放.

更多_output.yml 範例

簡單講,_output.yml放的是輸出到哪裡?gitbook? pdf? 或者是簡單html格式。

_output.yml 範例

bookdown::gitbook:
  css: style.css
  config:
    toc:
      before: |
        <li><a href="./">A Minimal Book Example</a></li>
      after: |
        <li><a href="https://github.com/rstudio/bookdown" target="blank">Published with bookdown</a></li>
    edit: https://github.com/rstudio/bookdown-demo/edit/master/%s
    download: ["pdf", "epub"]

# bookdown::pdf_book:
#   latex_engine: "xelatex"
#   keep_tex: true
#   includes:
#     in_header: header.tex

 bookdown::pdf_book:
   includes:
     in_header: latex/preamble.tex
     before_body: latex/before_body.tex
     after_body: latex/after_body.tex
   keep_tex: true
   dev: "cairo_pdf"
   latex_engine: xelatex
   citation_package: natbib
   pandoc_args: ["--top-level-division=chapter", "--lua-filter=latex/sidebar.lua"]
   template: null
   quote_footer: null
   toc_unnumbered: false
   number_sections: true    
---
output: 
  html_document: 
    highlight: tango
    number_sections: yes
    toc: yes
---

上面這段YML會被解譯到下面的R程式碼。 猜測:html_document對應rmarkdown::html_document
例如, number_sections: yes 會被當成html_document()的參數轉譯:

highlight: tango 對應到參數(沒有在下面)

rmarkdown::html_document( theme= "lumen",
                               template= template,
                               css= css,
                               toc= toc,
                               toc_float= TRUE,
                               toc_depth= 2,
                               number_sections= number_sections,
                               df_print = "paged",
                               code_folding = code_folding,
                               includes = includes(before_body = header)
                            )
}

knitr hook

顏色

設定警告訊息為紅色

knitr::knit_hooks$set(error = function(x, options) {
  paste0("<pre style=\"color: red;\"><code>", x, "</code></pre>")
})

其他範例

#Color Format
colFmt = function(x,color){
  outputFormat = opts_knit$get("rmarkdown.pandoc.to")
  if(outputFormat == 'latex')
    paste("\\textcolor{",color,"}{",x,"}",sep="")
  else if(outputFormat == 'html')
    paste("<font color='",color,"'>",x,"</font>",sep="")
  else
    x
}
